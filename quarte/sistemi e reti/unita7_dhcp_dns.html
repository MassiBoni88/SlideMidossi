Le Architetture N-tier basate su Client-Server

Lezione 1 · 01/03
📌 Nel contesto dei sistemi per il web i livelli prendono il nome di tier (strati) e le architetture si chiamano N-tier in base al numero N di strati progettati.

🏗️ I 3 Tier principali (lato server)

PRESENTATION Gestisce la modalità di comunicazione con i client: interfaccia grafica e rendering. È il front-end dell'applicazione web.
BUSINESS Elabora le richieste dei client e produce risultati. In questo strato risiedono le funzionalità dell'applicazione web.
DATA Gestisce l'accesso ai dati delle applicazioni web. È il back-end dell'applicazione web.

🖥️ Macchine server più utilizzate

Web Server
Presentation Tier — risponde alle richieste HTTP
Application Server
Business Tier — elabora le logiche applicative
DBMS Server
Data Tier — gestisce il database
Proxy Server
Presentation + Security Tier
AAA Server
Security Tier — Autenticazione, Autorizzazione, Accounting
💡 Sul lato client è sufficiente un unico tier: il Client Tier, rappresentato dal browser che gestisce le richieste HTTP.

Architetture 1-tier e 2-tier

Lezione 1 · 02/03

🖥️ Architettura 1-tier

Tutte le componenti (Presentation, Business, Data) fanno parte di un unico tier. Tipica dell'informatica centralizzata con terminali e sistemi mainframe (fino agli anni '90).

Scalabilità: difficoltà a modificare quantità o tipo di dati
Portabilità: difficoltà a spostare il sistema
Aggiornamento: difficoltà a migrare verso release nuove
Flessibilità: modifiche a un componente impattano tutto
ARCHITETTURA 1-TIER
Presentation Tier + Business Tier + Data Tier  (tutto insieme)
Terminal 1 · Terminal 2 · ... · Terminal N → File Server (mainframe)

🌐 Architettura 2-tier (Client-Server)

Modello Client-Server, sviluppato dagli anni '90, diventato il paradigma di Internet. Rende indipendente il Data Tier dai livelli Presentation e Business.

  • Il client non è più un terminale ma un host autonomo
  • Applicazioni e dati possono risiedere su macchine diverse
  • Ancora presenti problemi di scalabilità della 1-tier
ARCHITETTURA 2-TIER — LATO SERVER
Client Tier
(Host 1...N)
Presentation + Business Tier
Proxy Server → Web + App Server
Data Tier
DBMS Server
💡 L'architettura 2-tier è uno scenario tipico dell'informatica distribuita: applicazioni e dati possono risiedere in remoto, anche in CED aziendali o data center con server farm.

Architetture 3-tier, N-tier e Scalabilità

Lezione 1 · 03/03

🏗️ Architettura 3-tier

Si aggiunge un Middle Tier sempre sul modello Client-Server che rende indipendenti i 3 strati. Tipica degli anni Duemila, ogni strato è su macchine server differenti.

3-TIER
Client Tier
Proxy HTTP
Middle Tier
Web + App Server
Data Tier
DBMS Server

🔢 Architettura N-tier (Multi-Tier)

Architetture più attuali con diversi strati intermedi, uno per ogni funzione. Chiamate anche Multi-Tier Architecture.

Client
Web Server
AAA Server
App Server
DBMS

Microsoft inserisce i servizi di infrastruttura in Windows Server (AD DS, DHCP, DNS), le applicazioni nel .NET Framework, e propone Azure come architettura web cloud N-tier.

📈 Scalabilità orizzontale — Scale-out

Si aumenta il numero di nodi simili che ospitano l'applicazione. I nodi elaborano le richieste in parallelo.

PRO: fault-tolerance — il guasto di un nodo non blocca il servizio
CONTRO: il servizio deve supportare questo modello fin dalla progettazione

Esempio: sito web statico — raddoppiare i server raddoppia gli utenti servibili.

📊 Scalabilità verticale — Scale-up

Si aumentano le risorse di un singolo nodo: CPU più veloce, più RAM, più storage.

CONTRO: costi maggiori — hardware ad alte prestazioni è molto più costoso
USO TIPICO: tier del database, servizi cloud e CED locali
🔑 La scalabilità di un'applicazione è la capacità di aumentarne il throughput in proporzione all'hardware impiegato per ospitarla.

Le Soluzioni di Windows Server 2019

Lezione 2 · 01/02
🖥️ Windows Server 2019 è un sistema operativo predisposto per il cloud che supporta virtualizzazione, carichi di lavoro attuali e facilita la transizione verso il cloud computing. Si basa sulla precedente versione 2016 (prima versione cloud-ready).

📦 Le 3 edizioni

DATACENTER Per data center con alta percentuale di sistemi virtualizzati e ambienti cloud
STANDARD Per server fisici e ambienti enterprise con numero limitato di sistemi virtualizzati
ESSENTIALS Piccole imprese: max 25 utenti e 50 dispositivi — ideale per il cloud

🔧 Opzioni di installazione

  • Server Core (default): nessuna GUI, più veloce/sicuro, gestito via PowerShell o WAC da remoto
  • Desktop Experience: con interfaccia grafica Windows, maggior uso di risorse
  • Nano Server: molto piccolo, solo 64 bit, ideale per container — nessun accesso locale

✨ Novità principali di WS 2019

Cloud ibrido Connessione stretta con Azure: backup, recovery, aggiornamento dal cloud
Sicurezza Shielded VM (macchina virtuale schermata), Advanced Threat Protection, crittografia tra VM
Storage Storage Migration Service: passaggio semplificato a nuova versione OS o cloud
Container WSL migliorato — Windows Subsystem for Linux per eseguire Linux direttamente in Windows
System Insight Machine learning per previsioni su CPU, rete, storage — azioni correttive automatiche
🔗 La lista dei server role definisce il ruolo (compito) che ogni server deve svolgere nella rete. Progettare la rete significa conoscere e assegnare i server role corretti.

PowerShell e Windows Admin Center

Lezione 2 · 02/02

>_ PowerShell

Shell a riga di comando + linguaggio di scripting sviluppato da Microsoft. Open source dalla versione 6.0 (2016). Disponibile su Windows, Linux e macOS.

  • cmdlets: comandi specializzati per l'amministrazione (Registro di sistema, servizi, WMI)
  • Scripting simile a C#: sintassi coerente, output di un cmdlet come input di un altro
  • Esplorazione sistema: come navigazione file system, ma per il Registro di sistema
  • Interfaccia estensibile: produttori software possono aggiungere tool personalizzati
Get-Host | Select-Object Version
Show-Command
Add-WindowsFeature AD-Domain-Services

🌐 Windows Admin Center (WAC)

Nuova applicazione di gestione browser-based per Windows Server 2019. Gratuito, scaricabile separatamente dal sito Microsoft.

  • Gestisce server singoli (non sistemi su larga scala)
  • Funzionalità WAC-esclusive: Storage Migration, System Insights, SDN
  • Accesso alle console PowerShell e Remote Desktop

⚙️ Server Manager e RSAT

Strumenti RSAT (Remote Server Administration Tools) disponibili anche su Windows 10:

  • Server Manager: console di gestione multi-server, aggiunta ruoli e funzionalità, da WS 2008
  • MMC snap-in: Device Manager, Disk Management, Event Viewer, Local Users and Groups
  • Windows PowerShell cmdlets: gestione da riga di comando
📋 I 4 server role principali che vedremo nelle prossime lezioni: AD Domain Services · AD Users & Computers · DHCP Server · DNS Server

Il Domain Controller e Active Directory

Lezione 3 · 01/02
📌 Un controller di dominio (Domain Controller) permette di gestire le informazioni all'interno della rete in sicurezza: autenticazione degli utenti, condivisioni, Group Policy. È il vero cuore della rete.

🗂️ Active Directory Domain Services

AD DS è un raggruppamento logico di utenti e computer in un dominio gestito centralmente dai server Domain Controller.

Una struttura AD è un framework gerarchico di oggetti, suddivisi in 3 categorie:

Risorse
Stampanti, storage, condivisioni
Servizi
Email, applicazioni aziendali
Utenti
Account e gruppi

🔑 Concetti chiave di AD

  • Oggetto: singola entità (utente, computer, stampante, gruppo)
  • Attributi: caratteristiche dell'oggetto definite da uno Schema
  • Schema: definisce tipo di oggetti e relativi attributi ammessi
  • AD fornisce: informazioni, organizzazione, controllo accessi, sicurezza

🔧 Installazione AD DS — Server Manager

Procedura tramite Server Manager → Add Roles and Features Wizard:

1. Manager → Add Roles and Features
2. Role-based or feature-based installation
3. Server Roles → Active Directory Domain Services
4. Promote this server to a domain controller
5. Deployment Configuration → Add a new forest
💡 L'installazione tramite PowerShell:
Add-WindowsFeature AD-Domain-Services
consente installazioni locali e remote e di distribuire domini AD su più controller contemporaneamente.

Struttura di Active Directory

Lezione 3 · 02/02

🗂️ Livello logico

OU Organizational Unit: oggetti del dominio raggruppati per affinità (reparti, stampanti). Permette di applicare Group Policy Objects (GPO)
DOMINIO Gruppo di oggetti che condividono un directory database — identificato tramite namespace DNS (es. azienda.local)
ALBERO Albero di dominio: uno o più domini che condividono un namespace contiguo
FORESTA Foresta di dominio: insieme di uno o più alberi con catalogo globale, schema e struttura logica condivisi. È l'area totale in cui utenti, computer e oggetti sono accessibili

📍 Livello fisico

SITO Locazione geografica fisica che ospita reti. Usato per assegnare GPO, gestire replicazione AD e il traffico di collegamento tra sedi diverse. Contiene oggetti Subnet
SUBNET Specifico indirizzamento IP con una specifica subnet mask — livello fisico più granulare

🌍 Global Catalog

Database nel quale sono presenti tutti gli oggetti dei diversi domini presenti nella foresta, con una rappresentazione sintetica degli attributi di ogni oggetto.

Offre una visione globale di ogni oggetto in ogni dominio di ogni foresta — fondamentale in reti aziendali con più domini.

🏢 Nelle reti aziendali con sedi sparse in diverse città, i Siti e le Subnet diventano fondamentali per la comunicazione in rete e la gestione della replicazione.

Configurazione di Utenti e Computer

Lezione 4

👤 Active Directory Users and Computers

ADUC (snap-in MMC) permette di gestire gli oggetti del dominio: utenti, computer e gruppi.

Account Utente: username, password, profilo, cartelle home, orari di accesso, computer consentiti
Computer: ogni workstation deve essere aggiunta al dominio per usufruire dei servizi AD

👥 I Gruppi in Active Directory

Security group: controllo accesso a risorse (cartelle, stampanti, applicazioni)
Distribution group: liste di distribuzione email (non gestisce permessi)
Scope dei gruppi:
Local
Solo sul DC locale
Global
In tutto il dominio
Universal
In tutta la foresta

📋 Group Policy Objects (GPO)

Politiche di gestione centralizzata dell'azienda, applicabili a dominio, OU o Sito. Gestite tramite Group Policy Management Console (GPMC).

Esempi di GPO:
  • Limitazioni di accesso ad applicazioni o siti
  • Script eseguiti all'avvio/spegnimento del computer
  • Configurazione automatica del desktop e del browser
  • Gestione delle stampanti e delle condivisioni
  • Restrizioni sull'uso di dispositivi rimovibili (USB)
🔐 Le GPO vengono applicate in cascata: prima le policy del sito, poi del dominio, poi dell'OU. Le policy dell'OU hanno priorità sulle altre. È possibile bloccare l'ereditarietà.
💡 Ogni persona che usa computer all'interno del dominio riceve un account univoco con credenziali e grado di accesso definiti dall'amministratore di rete.

Il Servizio DHCP Server

Lezione 5 · 01/02
📌 DHCP (Dynamic Host Configuration Protocol) è un servizio che assegna automaticamente la configurazione IP a ogni host che si connette alla rete, evitando la configurazione manuale.

📡 Parametri assegnati da DHCP

Indirizzo IP
Assegnato dal pool dello Scope per la durata del lease
Subnet Mask
Identifica la rete di appartenenza
Default Gateway
Indirizzo del router per l'uscita dalla rete locale
DNS Server
Indirizzi del/dei DNS per la risoluzione dei nomi
Lease Time
Durata del "prestito" dell'indirizzo IP (es. 8 ore)

⚙️ Configurazione DHCP su WS 2019

SCOPE Pool di indirizzi IP disponibili per l'assegnazione (es. 192.168.1.100–200)
EXCLUSION Range di indirizzi esclusi dallo Scope (server, router, stampanti con IP fisso)
RESERVATION IP fisso assegnato a uno specifico indirizzo MAC — il dispositivo riceve sempre lo stesso IP
AUTORIZZAZIONE Il server DHCP deve essere autorizzato in Active Directory per funzionare (sicurezza anti-rogue DHCP)
🔄 Il processo DHCP segue 4 fasi: DISCOVER (client cerca server) → OFFER (server propone IP) → REQUEST (client accetta) → ACK (server conferma il lease).

Il Servizio DNS Server

Lezione 5 · 02/02
📌 DNS (Domain Name System) è il sistema che traduce i nomi di dominio (es. www.azienda.it) in indirizzi IP e viceversa — indispensabile per Active Directory e per la navigazione web.

🗂️ Zone DNS

FORWARD Forward Lookup Zone: risolve nome → indirizzo IP (risoluzione diretta). La più comune.
REVERSE Reverse Lookup Zone: risolve IP → nome host (risoluzione inversa). Usata per diagnostica e log.
AD-INT. Zone integrate in Active Directory: replicate automaticamente tra tutti i DC — più sicure e resilienti

📋 Record DNS principali

A
Nome host → indirizzo IPv4
AAAA
Nome host → indirizzo IPv6
CNAME
Alias per un altro nome (Canonical Name)
MX
Server di posta elettronica del dominio
NS
Name Server autoritativo per la zona
PTR
Indirizzo IP → nome host (reverse)
🔗 Il DNS è indispensabile per Active Directory: ogni Domain Controller registra i propri servizi nel DNS tramite record SRV, permettendo ai client di trovare i controller del dominio.

Riepilogo — Unità 7

L1 — Architetture N-tier

  • 3 tier fondamentali: Presentation (front-end), Business (logica), Data (back-end)
  • Evoluzione: 1-tier (mainframe) → 2-tier (Client-Server) → 3-tier (Middle Tier) → N-tier
  • Scalabilità orizzontale (scale-out, fault-tolerance) vs verticale (scale-up, costi)
  • Microsoft: servizi in WS 2019, app in .NET, web cloud in Azure

L2 — Windows Server 2019

  • 3 edizioni: Datacenter (cloud), Standard (enterprise), Essentials (PMI)
  • 3 installazioni: Server Core (default, no GUI), Desktop Experience, Nano Server
  • PowerShell: cmdlets, scripting multipiattaforma
  • WAC: gestione browser-based · Server Manager: multi-server management

L3+L4 — Domain Controller e Utenti

  • Domain Controller: autenticazione, condivisioni, Group Policy — cuore della rete
  • AD DS: oggetti → risorse, servizi, utenti con attributi definiti da uno Schema
  • Struttura logica: OU → Dominio → Albero → Foresta · fisica: Sito, Subnet
  • GPO: politiche centralizzate per dominio, OU, sito
  • Gruppi: Security (permessi) e Distribution (email)

L5 — DHCP e DNS

  • DHCP: IP, Subnet, Gateway, DNS assegnati automaticamente · Scope, Exclusion, Reservation
  • DHCP deve essere autorizzato in AD · fasi: DISCOVER → OFFER → REQUEST → ACK
  • DNS: nome → IP (Forward Zone) e IP → nome (Reverse Zone)
  • Record: A (IPv4), AAAA (IPv6), CNAME (alias), MX (mail), PTR (reverse)
  • DNS integrato in AD: zone replicate automaticamente tra Domain Controller
