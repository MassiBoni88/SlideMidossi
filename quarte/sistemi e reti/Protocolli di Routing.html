Routing — Concetti Base

L1 · 03/15

Routing statico

Le route sono configurate manualmente dall'amministratore. Non si adattano ai cambiamenti della rete. Adatto a reti piccole o per route predefinite (default route).

ip route 192.168.2.0 255.255.255.0 10.0.0.2
ip route 0.0.0.0 0.0.0.0 10.0.0.1 ! default

Routing dinamico

Il router scambia informazioni di routing con i router vicini tramite un protocollo. Le route si aggiornano automaticamente in caso di guasti o modifiche alla topologia.

  • IGP (Interior Gateway Protocol): routing dentro un AS — RIP, OSPF, EIGRP
  • EGP (Exterior Gateway Protocol): routing tra AS diversi — BGP

Administrative Distance (AD)

Indica la affidabilità della sorgente di una route. Se esistono più route per la stessa destinazione, vince quella con AD più bassa.

Distance Vector — Algoritmo e Problemi

L2 · 04/15

Principio Distance Vector

Ogni router conosce solo la distanza (metrica) verso le destinazioni e la direzione (next-hop) per raggiungerle. Non ha una visione completa della topologia.

  • Basato sull'algoritmo di Bellman-Ford
  • I router inviano periodicamente l'intera tabella di routing ai vicini diretti
  • Ogni router aggiorna la propria tabella prendendo il percorso con metrica minore

Problema del Count-to-Infinity

Se un link cade, i router continuano ad incrementare la metrica pensando che la rete sia ancora raggiungibile tramite un vicino che in realtà la conosce da loro. La metrica cresce all'infinito prima della convergenza.

Soluzioni al Count-to-Infinity

  • Maximum hop count: RIP considera ∞ = 16 hop, blocca i loop
  • Split Horizon: non si annuncia una route all'interfaccia da cui è stata appresa
  • Split Horizon con Poison Reverse: si annuncia la route con metrica ∞ all'interfaccia da cui è appresa
  • Triggered Update: invio immediato degli aggiornamenti quando una route cambia (non si aspetta il timer periodico)
  • Hold-down Timer: dopo aver ricevuto un'informazione di guasto, il router ignora gli aggiornamenti per quella route per un certo periodo
⚠️ La convergenza lenta è il principale svantaggio dei protocolli Distance Vector: in reti grandi possono volerci minuti per stabilizzarsi dopo un guasto.

RIP — Routing Information Protocol

L2 · 05/15

Caratteristiche RIP

  • Metrica: hop count (numero di router attraversati)
  • Massimo: 15 hop — rete con 16 hop è considerata irraggiungibile
  • Aggiornamenti: broadcast/multicast ogni 30 secondi (tutta la tabella)
  • RIPv1: classful (no VLSM, no CIDR), broadcast 255.255.255.255
  • RIPv2: classless (VLSM supportato), multicast 224.0.0.9, autenticazione MD5
  • Timer: update 30 s · invalid 180 s · flush 240 s · holddown 180 s

Configurazione Cisco — RIPv2

Router(config)# router rip
Router(config-router)# version 2
Router(config-router)# network 192.168.1.0
Router(config-router)# network 10.0.0.0
Router(config-router)# no auto-summary

! Verifica
Router# show ip route
Router# show ip protocols
Router# debug ip rip

Link State — Algoritmo SPF

L3 · 06/15

Principio Link State

Ogni router costruisce una mappa completa della topologia di rete (LSDB — Link State Database) e calcola autonomamente il percorso più breve verso ogni destinazione con l'algoritmo di Dijkstra (SPF — Shortest Path First).

  • I router si scambiano LSA (Link State Advertisement) — non l'intera tabella
  • Gli LSA vengono inondati (flooded) a tutta la rete → tutti hanno la stessa LSDB
  • Convergenza molto più rapida rispetto al Distance Vector

Fasi di funzionamento OSPF

  • 1 — Hello: i router si scoprono scambiando Hello packet (multicast 224.0.0.5); stabiliscono le adiacenze (adjacency)
  • 2 — Scambio LSA: ogni router annuncia i propri link (stato, costo) → popola la LSDB
  • 3 — SPF: ogni router esegue Dijkstra sulla LSDB → calcola l'albero dei cammini minimi
  • 4 — Tabella di routing: le rotte migliori vengono installate nella RIB

Costo OSPF

Il costo di un'interfaccia è calcolato come:

OSPF — Aree e Configurazione Cisco

L3 · 07/15

Architettura a due livelli

OSPF suddivide la rete in aree per ridurre il traffico LSA e la complessità dell'algoritmo SPF. Ogni area ha la propria LSDB.

  • Area 0 (Backbone): obbligatoria, tutte le aree devono connettersi ad essa
  • Area non-backbone: area interna connessa al backbone
  • ABR (Area Border Router): router con interfacce in più aree
  • ASBR (AS Boundary Router): redistribuisce route esterne in OSPF
  • DR / BDR: Designated/Backup DR eletti sui segmenti multi-access (Ethernet) per ridurre le adiacenze da n² a n

Configurazione Cisco — OSPFv2

Router(config)# router ospf 1
Router(config-router)# router-id 1.1.1.1
Router(config-router)# network 192.168.1.0 0.0.0.255 area 0
Router(config-router)# network 10.0.0.0 0.0.0.3 area 0
Router(config-router)# auto-cost reference-bandwidth 10000

! Costo manuale su interfaccia
Router(config-if)# ip ospf cost 10

! Verifica
Router# show ip ospf neighbor
Router# show ip ospf database
Router# show ip route ospf

Stati di adiacenza OSPF

  • Down → Init: ricevuto Hello da un vicino
  • 2-Way: il vicino ha visto il nostro Router-ID nel suo Hello → adiacenza bidirezionale (DR/BDR election)
  • ExStart / Exchange: scambio Database Description (DBD)
  • Loading: richiesta LSR e ricezione LSU
  • Full: LSDB sincronizzata → adiacenza completa
✅ OSPF è il protocollo IGP più diffuso in reti enterprise. Supporta VLSM, convergenza rapida (secondi), nessun limite di hop, autenticazione MD5/SHA e IPv6 (OSPFv3).

BGP — Border Gateway Protocol

L4 · 08/15

Cos'è BGP

BGP è l'unico protocollo EGP usato su Internet: gestisce il routing tra Autonomous System (AS) diversi. Un AS è un insieme di reti sotto un'unica amministrazione con una politica di routing comune, identificato da un numero ASN (16 o 32 bit, assegnato da IANA/RIR).

  • iBGP (internal BGP): sessione BGP tra router dello stesso AS
  • eBGP (external BGP): sessione BGP tra router di AS diversi
  • Usa TCP porta 179 (connessione affidabile)
  • Non usa la metrica tradizionale: sceglie il percorso in base agli attributi del path

Principali attributi BGP

  • AS_PATH: lista degli AS attraversati — usato per prevenire loop e scegliere il percorso più corto
  • NEXT_HOP: indirizzo IP del prossimo router da raggiungere
  • LOCAL_PREF: preferenza locale, usato in iBGP per scegliere il punto di uscita dall'AS (valore più alto = preferito)
  • MED (Multi Exit Discriminator): suggerisce all'AS adiacente quale link preferire per entrare nel nostro AS (valore più basso = preferito)
  • WEIGHT: attributo Cisco locale al router, non propagato (valore più alto = preferito)
  • Community: tag arbitrario per applicare politiche di routing in gruppi
🌐 BGP è il protocollo che tiene insieme Internet. I provider (ISP, CDN, grandi aziende) usano BGP per annunciare i propri prefissi IP al mondo. Un errore di configurazione BGP può causare blackout globali (es. Facebook 2021).

BGP — Selezione del Percorso e Configurazione

L4 · 09/15

Ordine di selezione del best path BGP

In caso di più percorsi verso la stessa destinazione BGP sceglie nell'ordine:

  • 1. WEIGHT più alto (Cisco locale)
  • 2. LOCAL_PREF più alto
  • 3. Route originate localmente (network / redistribute)
  • 4. AS_PATH più corto
  • 5. Tipo di origine: IGP < EGP < Incomplete
  • 6. MED più basso
  • 7. eBGP preferito su iBGP
  • 8. IGP metric più bassa verso il NEXT_HOP

Configurazione Cisco — eBGP

! Router AS 65001, neighbor in AS 65002
Router(config)# router bgp 65001
Router(config-router)# bgp router-id 1.1.1.1
Router(config-router)# neighbor 203.0.113.2 remote-as 65002
Router(config-router)# network 10.0.0.0 mask 255.0.0.0

! iBGP (stesso AS)
Router(config-router)# neighbor 192.168.1.2 remote-as 65001
Router(config-router)# neighbor 192.168.1.2 update-source Loopback0

! Verifica
Router# show ip bgp summary
Router# show ip bgp
Router# show ip bgp neighbors
💡 In iBGP il next-hop non viene modificato automaticamente — occorre neighbor X update-source Loopback0 e next-hop-self per garantire la raggiungibilità del next-hop all'interno dell'AS.

EIGRP — Enhanced Interior Gateway Routing Protocol

L5 · 10/15

Caratteristiche EIGRP

EIGRP è un protocollo ibrido (Distance Vector avanzato) sviluppato da Cisco. Combina vantaggi di Distance Vector e Link State:

  • Invia aggiornamenti solo al cambio di topologia (non periodici)
  • Mantiene una tabella dei vicini (neighbor table) e una tabella della topologia (topology table)
  • Convergenza molto rapida grazie all'algoritmo DUAL
  • Supporta VLSM, IPv4 e IPv6
  • AD interna: 90 · AD esterna: 170
  • Multicast 224.0.0.10 per gli Hello packet

Algoritmo DUAL

Diffusing Update Algorithm: garantisce loop-freedom e convergenza rapida. Per ogni rete EIGRP identifica:

  • Successor: il router con il percorso migliore verso la destinazione (installato nella tabella di routing)
  • Feasible Successor: percorso di backup garantito loop-free, pronto per essere usato immediatamente in caso di guasto del Successor senza ricalcolare
  • Feasible Distance (FD): metrica totale del percorso migliore
  • Reported Distance (RD): metrica annunciata dal vicino verso la destinazione

Metrica composita EIGRP

La metrica è calcolata in base a 5 parametri (K-values). Per default K1=1, K2=0, K3=1, K4=0, K5=0 → solo Bandwidth e Delay:

Metrica = 256 × (107/BW + Delay/10)

BW = bandwidth minima sul path (kbps) · Delay = somma dei delay (decine di µs)

Configurazione Cisco — EIGRP

Router(config)# router eigrp 100
Router(config-router)# network 192.168.1.0 0.0.0.255
Router(config-router)# network 10.0.0.0 0.0.0.3
Router(config-router)# no auto-summary

! Verifica
Router# show ip eigrp neighbors
Router# show ip eigrp topology
Router# show ip route eigrp

Redistribuzione tra Protocolli

L5 · 11/15

Cos'è la redistribuzione

La redistribuzione consente a un router che esegue più protocolli di routing di condividere le route apprese da un protocollo con un altro. Necessaria in reti eterogenee o durante migrazioni.

  • Il router di redistribuzione (ASBR in OSPF) si trova al confine tra due domini di routing
  • Le route ridistribuite appaiono come route esterne nel protocollo di destinazione (es. O E2 in OSPF)
  • Occorre specificare una metrica seed per le route ridistribuite

Configurazione — da RIP a OSPF

! Sul router di redistribuzione
Router(config)# router ospf 1
Router(config-router)# redistribute rip metric 20 subnets

Router(config)# router rip
Router(config-router)# redistribute ospf 1 metric 5

Route Map — Filtraggio avanzato

Le route-map permettono di controllare quali route redistribuire e con quali attributi. Simili alle ACL ma con più opzioni di modifica.

! Redistribuisce solo le rotte 10.x.x.x da RIP a OSPF
Router(config)# access-list 10 permit 10.0.0.0 0.255.255.255
Router(config)# route-map RIP_TO_OSPF permit 10
Router(config-route-map)# match ip address 10
Router(config-route-map)# set metric 50

Router(config)# router ospf 1
Router(config-router)# redistribute rip route-map RIP_TO_OSPF subnets
⚠️ La redistribuzione bidirezionale tra due protocolli può causare loop di routing. Usare sempre route-map o distribute-list per filtrare e controllare cosa viene redistribuito in ciascuna direzione.

Routing IPv6

06 · 13/15

Protocolli IPv6

  • OSPFv3: versione di OSPF per IPv6. Usa indirizzi link-local come next-hop; le aree e il meccanismo SPF restano invariati. Annuncia prefissi IPv6 anziché IPv4.
  • RIPng (RIP next generation): RIP adattato per IPv6. Usa multicast FF02::9. Stessi limiti di RIPv2 (max 15 hop).
  • EIGRPv6: versione IPv6 di EIGRP, stesso algoritmo DUAL, configurata direttamente sull'interfaccia.
  • BGP4+ (MP-BGP): estensione multiprotocollo di BGP, trasporta NLRI sia IPv4 sia IPv6 nella stessa sessione TCP.

Configurazione OSPFv3 Cisco

! Abilita routing IPv6 (globale)
Router(config)# ipv6 unicast-routing

Router(config)# ipv6 router ospf 1
Router(config-rtr)# router-id 1.1.1.1

! Applicato sull'interfaccia (non network command)
Router(config-if)# ipv6 ospf 1 area 0

! Verifica
Router# show ipv6 ospf neighbor
Router# show ipv6 route ospf

Riepilogo — Unità 3

🟢 L1 — Routing Base

  • Statico: manuale, stabile. Dinamico: automatico, adattivo
  • AD: affidabilità della sorgente (0=connesso, 1=statico, 90=EIGRP, 110=OSPF, 120=RIP)
  • IGP (dentro l'AS) vs EGP (tra AS). Letture tabella: C/S/R/O/D/B

🔴 L2 — RIP

  • Bellman-Ford, hop count, max 15 hop (16 = irraggiungibile)
  • RIPv1 classful (broadcast) · RIPv2 classless (multicast 224.0.0.9, MD5)
  • Convergenza lenta, count-to-infinity → split horizon, triggered update

🔵 L3 — OSPF

  • Dijkstra SPF su LSDB completa; costo = 108/bandwidth
  • Area backbone (area 0) obbligatoria; ABR connette le aree
  • DR/BDR su segmenti multi-access; adiacenza Full = LSDB sincronizzata
  • Comandi chiave: router ospf 1, network X Y area Z, router-id

🟡 L4 — BGP

  • Unico EGP; collega AS diversi tramite TCP 179
  • eBGP (tra AS) vs iBGP (stesso AS)
  • Selezione: Weight → LOCAL_PREF → origine locale → AS_PATH corto → MED basso

🔴 L5 — EIGRP

  • Ibrido Cisco; DUAL garantisce loop-freedom e convergenza rapida
  • Successor (percorso attivo) + Feasible Successor (backup immediato)
  • Metrica = 256×(107/BW + Delay/10); aggiornamenti solo al cambio topologia

🔵 IPv6 e Redistribuzione

  • OSPFv3 (config su interfaccia), RIPng, EIGRPv6, MP-BGP
  • Redistribuzione: redistribute + metrica seed + subnets
  • Route-map: filtra e modifica le route redistribuite — evita loop
