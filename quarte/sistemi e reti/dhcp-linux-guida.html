Configurazione DHCP & IP Statico

Guida pratica alla configurazione della rete su Linux Server (Ubuntu)

Configurare un Server DHCP

Installazione e configurazione di isc-dhcp-server su Ubuntu Server per assegnare automaticamente gli indirizzi IP ai client della rete.

1
Installazione del pacchetto
apt install isc-dhcp-server

Prima di tutto si aggiorna la lista dei pacchetti e si installa il server DHCP:

BASH — terminale 
sudo apt update
sudo apt install isc-dhcp-server
2
Configurazione principale
/etc/dhcp/dhcpd.conf

Questo è il file principale dove si definisce il pool di indirizzi, il gateway, i DNS e la durata dei lease:

/etc/dhcp/dhcpd.conf 
# Durata default del lease: 10 minuti
default-lease-time 600;
# Durata massima del lease: 2 ore
max-lease-time 7200;

# Definizione della subnet
subnet 192.168.1.0 netmask 255.255.255.0 {
    # Pool: indirizzi assegnabili ai client
    range 192.168.1.100 192.168.1.200;
    # Gateway predefinito
    option routers 192.168.1.1;
    # Server DNS
    option domain-name-servers 8.8.8.8, 8.8.4.4;
    # Indirizzo di broadcast
    option broadcast-address 192.168.1.255;
}
Lease: è il tempo per cui un indirizzo IP è "affittato" a un client. Alla scadenza il client deve rinnovarlo, altrimenti l'IP torna disponibile nel pool.
3
Specificare l'interfaccia di rete
/etc/default/isc-dhcp-server

Bisogna indicare su quale interfaccia il server deve ascoltare le richieste dei client:

/etc/default/isc-dhcp-server 
INTERFACESv4="enp0s3"  # sostituire con il nome reale dell'interfaccia
Nota: per trovare il nome della tua interfaccia usa il comando ip a o ifconfig. I nomi comuni sono eth0, enp0s3, ens33.
4
Avviare e abilitare il servizio
systemctl start / enable

Si avvia il servizio e lo si abilita per il riavvio automatico del server:

BASH — terminale 
# Avvia subito il servizio
sudo systemctl start isc-dhcp-server

# Abilita l'avvio automatico al boot
sudo systemctl enable isc-dhcp-server

# Verifica lo stato
sudo systemctl status isc-dhcp-server
5
Assegnazione statica per MAC address
opzionale — come in BOOTP

Se vuoi che una specifica macchina riceva sempre lo stesso IP (ad esempio un server o una stampante), aggiungi questa sezione in /etc/dhcp/dhcpd.conf:

/etc/dhcp/dhcpd.conf — aggiunta 
host pc-laboratorio {
    hardware ethernet AA:BB:CC:DD:EE:01;
    fixed-address 192.168.1.50;
}
Differenza con BOOTP: anche con l'assegnazione fissa, gli altri client non registrati ricevono comunque un IP dal pool in modo automatico. Non è necessario registrare ogni dispositivo.
📁 Riepilogo file coinvolti
FileScopo
/etc/dhcp/dhcpd.confConfigurazione principale (pool, lease, DNS…)
/etc/default/isc-dhcp-serverInterfaccia di rete su cui ascoltare
systemctlGestione del servizio (start/stop/enable)

Configurare un IP Statico (Client)

Su Ubuntu Server moderno la rete si configura tramite Netplan. Invece di ricevere un IP dal server DHCP, si assegna manualmente un indirizzo fisso al proprio sistema.

1
Trovare il file di configurazione Netplan
/etc/netplan/

Netplan legge i file di configurazione dalla directory /etc/netplan/. Elencali con:

BASH — terminale 
ls /etc/netplan/
# output tipico: 00-installer-config.yaml
2
Modificare il file YAML
/etc/netplan/00-installer-config.yaml

Apri il file con un editor e imposta la configurazione statica. Esempio completo:

/etc/netplan/00-installer-config.yaml 
network:
  version: 2
  ethernets:
    enp0s3:                        # nome interfaccia
      dhcp4: no                    # disabilita DHCP
      addresses:
        - 192.168.1.50/24          # IP statico / subnet mask
      routes:
        - to: default
          via: 192.168.1.1         # gateway predefinito
      nameservers:
        addresses:
          - 8.8.8.8                 # DNS primario
          - 8.8.4.4                 # DNS secondario
Attenzione: il formato YAML è sensibile all'indentazione. Usa sempre spazi, mai TAB.
3
Applicare la configurazione
netplan apply

Prima si testa la configurazione per verificare che non ci siano errori, poi la si applica:

BASH — terminale 
# Testa senza applicare (sicuro)
sudo netplan try

# Applica definitivamente
sudo netplan apply
4
Verificare la configurazione
ip a / ping

Controlla che l'indirizzo IP sia stato assegnato correttamente:

BASH — terminale 
# Mostra gli indirizzi IP delle interfacce
ip a

# Testa la connettività verso il gateway
ping 192.168.1.1

# Testa la connettività verso internet
ping 8.8.8.8
⚖️ DHCP vs IP Statico — quando usare quale
ScenarioSoluzione consigliata
PC utenti, laptop, smartphoneDHCP (automatico)
Server, router, stampanti di reteIP Statico
Il server DHCP stessoIP Statico (deve essere sempre raggiungibile)
Dispositivi IoT fissiIP Statico o prenotazione DHCP
