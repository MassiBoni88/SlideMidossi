Struttura di un Programma C++

Sintassi · 01/02
📌 Ogni programma C++ è composto da istruzioni organizzate in funzioni. Il punto di ingresso obbligatorio è la funzione main(): il sistema operativo la chiama per avviare il programma. 
#include <iostream>
using namespace std;

int main() {
    // corpo del programma
    cout << "Ciao, mondo!" << endl;
    return 0;
}

⚙️ Regole fondamentali della sintassi

  • Ogni istruzione termina con punto e virgola ;
  • I blocchi di codice sono delimitati da parentesi graffe { }
  • Il C++ è case-sensitive: Main ≠ main
  • Gli spazi e le righe vuote non influenzano l'esecuzione
  • Il codice viene compilato prima di essere eseguito

🔍 Anatomia del programma

#include Direttiva al preprocessore: include una libreria esterna nel programma
using Permette di usare nomi dello spazio std senza il prefisso std::
int main() Funzione principale — restituisce un intero al sistema operativo (0 = successo)
return 0 Valore restituito al SO: 0 indica terminazione corretta del programma
⚠️ Un programma C++ deve essere compilato (es. con g++ programma.cpp -o programma) per ottenere un file eseguibile. Errori di sintassi bloccano la compilazione.

📂 Estensione dei file

I file sorgente C++ usano estensione .cpp (o .cxx, .cc). I file header usano .h o .hpp. Il compilatore trasforma il sorgente in codice macchina eseguibile.

Namespace e Processo di Compilazione

Sintassi · 02/02

🗂️ Cos'è un namespace

Un namespace è uno spazio di nomi che raggruppa identificatori per evitare conflitti. La libreria standard C++ usa il namespace std.

// Senza using namespace std:
std::cout << "Ciao" << std::endl;

// Con using namespace std:
using namespace std;
cout << "Ciao" << endl;

Nei progetti grandi si preferisce scrivere std:: esplicitamente per chiarezza.

💡 I namespace risolvono il problema delle collisioni di nomi: se due librerie definiscono una funzione sort(), i namespace le distinguono senza ambiguità.

Le Librerie Standard del C++

Librerie · 01/01
📦 Una libreria è una raccolta di funzioni, classi e costanti già pronte all'uso. Si include con la direttiva #include seguita dal nome del file header tra <> (librerie standard) o " " (file propri).

📚 Librerie standard principali

<iostream> Input/Output: cout, cin, cerr, endl
<string> Gestione delle stringhe di testo: classe string con metodi come .length(), .substr()
<cmath> Funzioni matematiche: sqrt(), pow(), abs(), ceil(), floor()
<vector> Array dinamici: vector<T> con dimensione variabile a runtime
<algorithm> Algoritmi su collezioni: sort(), find(), min(), max()
<fstream> Lettura e scrittura su file: ifstream (lettura) e ofstream (scrittura)
<cstdlib> Utilità generali: rand(), srand(), exit(), conversioni

📝 Sintassi #include

// Librerie standard (tra parentesi angolari)
#include <iostream>
#include <string>
#include <cmath>

// File header propri (tra virgolette)
#include "mia_libreria.h"
#include "utils/helper.hpp"

🔗 Come funziona #include

Il preprocessore sostituisce letteralmente la direttiva #include con il contenuto del file header indicato, prima che il compilatore analizzi il codice.

💡 I file header (.h / .hpp) contengono le dichiarazioni delle funzioni. Il codice reale (implementazione) si trova nei file .cpp o nelle librerie già compilate.
🚀 In C++ moderno (C++17/20) si può includere praticamente tutto con #include <bits/stdc++.h>, usato spesso nelle competizioni di programmazione (non consigliato in produzione).

L'Output con cout

Input/Output · 01/02
📺 cout (Console OUTput) è l'oggetto standard per stampare dati sullo schermo. Fa parte della libreria <iostream> e usa l'operatore di inserimento << per inviare dati al flusso di output.

🖨️ Esempi di output

#include <iostream>
using namespace std;

int main() {
    // Testo semplice
    cout << "Ciao, mondo!" << endl;

    // Variabili
    int eta = 20;
    cout << "Età: " << eta << endl;

    // Concatenazione con <<
    cout << "A=" << 3 << " B=" << 7 << endl;

    // Senza andare a capo (\n)
    cout << "Prima. ";
    cout << "Seconda.\n";
    return 0;
}

↩️ endl vs \n

Simbolo
Effetto
Differenza
endl
Va a capo
Svuota il buffer (flush)
\n
Va a capo
Più veloce, no flush

Preferire \n per le prestazioni; usare endl solo quando serve svuotare il buffer (es. output immediato).

💡 Si possono concatenare più << in una sola riga: cout << "x=" << x << " y=" << y << "\n"; — il C++ valuta da sinistra a destra.

L'Input con cin

Input/Output · 02/02
⌨️ cin (Console INput) legge dati inseriti dall'utente tramite tastiera. Usa l'operatore di estrazione >> per leggere un valore e memorizzarlo in una variabile. Salta automaticamente spazi e invii. 
#include <iostream>
#include <string>
using namespace std;

int main() {
    int eta;
    cout << "Inserisci la tua età: ";
    cin >> eta;
    cout << "Hai " << eta << " anni." << endl;

    string nome;
    cout << "Il tuo nome: ";
    cin >> nome;           // legge una parola
    cout << "Ciao, " << nome << "!" << endl;

    return 0;
}

📋 cin vs getline

cin >> Legge fino al primo spazio/invio. Usato per interi, float, singole parole
getline Legge l'intera riga inclusi gli spazi. Necessario per frasi complete 
string frase;
// Legge tutta la riga con spazi
getline(cin, frase);
cout << "Hai scritto: " << frase << "\n";

I Commenti nel Codice

Commenti · 01/01
📝 I commenti sono porzioni di testo ignorate completamente dal compilatore. Servono a documentare il codice, spiegarne il funzionamento e migliorarne la leggibilità per chi lo legge (o per se stessi in futuro).

// Commento su riga singola

Tutto ciò che segue // sulla stessa riga viene ignorato dal compilatore.

int main() {
    // Questo è un commento su riga singola
    int x = 10;  // commento a fine riga

    // x = 20; <-- questa riga è "commentata out"
    cout << x << endl;
    return 0;
}

/* */ Commento multiriga

Tutto ciò che è racchiuso tra /* e */ viene ignorato, anche su più righe.

/*
  Programma: Calcolatrice semplice
  Autore: Mario Rossi
  Data: 01/05/2026
  Versione: 1.0
*/

int somma(int a, int b) {
    /* restituisce la somma
       dei due parametri */
    return a + b;
}

✅ Buone pratiche

✓ Commenta il perché, non il cosa (il codice già dice cosa fa)
✓ Documenta le funzioni con parametri, valore di ritorno e scopo
✓ Usa // per spiegazioni brevi inline, /* */ per intestazioni e blocchi
✗ Evita commenti ovvi come // incrementa x di 1 prima di x++;
✗ Non lasciare grandi blocchi di codice commentato nel sorgente finale

🔧 Commentare per disabilitare codice

Utile durante il debug per escludere temporaneamente parti di codice:

int risultato = a + b;
// risultato = a * b;  // alternativa disabilitata
cout << risultato;
⚠️ I commenti /* */ non possono essere annidati: /* /* testo */ */ causa un errore di compilazione. Per blocchi annidati usare più righe con //.

Le Variabili — Dichiarazione e Inizializzazione

Variabili · 01/03
📦 Una variabile è uno spazio di memoria a cui si assegna un nome simbolico. Contiene un valore che può cambiare durante l'esecuzione. In C++ ogni variabile deve essere dichiarata prima di essere usata, specificando il suo tipo.

📝 Dichiarazione e inizializzazione

// Dichiarazione (senza valore iniziale)
int eta;

// Dichiarazione + inizializzazione
int anno = 2026;
double pi = 3.14159;
char lettera = 'A';
bool attivo = true;
string nome = "Mario";

// Più variabili dello stesso tipo
int x = 1, y = 2, z = 3;

📏 Regole per i nomi delle variabili

✓ Possono contenere lettere, cifre e underscore _
✓ Devono iniziare con una lettera o un underscore (non una cifra)
✓ Sono case-sensitive: Eta, eta e ETA sono tre variabili distinte
✗ Non possono essere parole chiave riservate: int, for, while…
✗ Non possono contenere spazi o caratteri speciali (-, @, .…)

🎨 Convenzioni di nomenclatura

camelCase etaUtente, nomeCompleto — variabili locali
snake_case eta_utente, nome_completo — stile C
MAIUSCOLO MAX_VALORE, PI_GRECO — costanti
⚠️ Una variabile dichiarata senza inizializzazione contiene un valore indeterminato (garbage). Usarla prima di assegnarle un valore è un errore logico che può causare comportamenti imprevedibili.

I Tipi di Dato Fondamentali

Variabili · 02/03
🔬 Il tipo di una variabile determina: quanta memoria occupa, quali valori può contenere e quali operazioni si possono eseguire su di essa. Il compilatore C++ controlla i tipi a compile-time (type safety).

📊 Tipi interi

TipoDimensioneIntervallo

short2 byte−32.768 … 32.767
long4/8 byte±2·10⁹ / ±9·10¹⁸
long long8 byte±9·10¹⁸
unsigned int4 byte0 … 4·10⁹

📊 Tipi floating point

float4 byte~7 cifre decimali
double8 byte~15 cifre decimali
long double10/16 byte~18 cifre decimali

Si preferisce sempre double a float per la maggiore precisione.

📊 Altri tipi fondamentali

TipoDimensioneValori / Uso
char1 byte'A' … 'z', '0'…'9', simboli
bool1 bytetrue (1) oppure false (0)
stringvariabileTesto (dalla libreria <string>)
void0 byteNessun tipo (usato per funzioni)

💻 Esempi pratici

int    punteggio = 100;
double media     = 8.75;
char   voto      = 'A';
bool   promosso  = true;
string studente  = "Giulia";

// sizeof() restituisce i byte occupati
cout << sizeof(int)    << endl; // 4
cout << sizeof(double) << endl; // 8
cout << sizeof(char)   << endl; // 1
💡 Con auto (C++11) il compilatore deduce automaticamente il tipo dalla costante di inizializzazione: auto x = 3.14; deduce double.

Le Costanti

Variabili · 03/03
🔒 Una costante è una variabile il cui valore non può cambiare dopo l'inizializzazione. Si dichiara con la parola chiave const prima del tipo. Qualsiasi tentativo di modifica viene segnalato come errore dal compilatore.

🔒 Dichiarazione con const

La parola chiave const va posta prima del tipo. La costante deve essere inizializzata subito alla dichiarazione.

// Costante intera
const int MAX_STUDENTI = 30;

// Costante reale
const double PI = 3.14159265;

// Costante stringa
const string SCUOLA = "ITI Informatica";

// PI = 3.0; <-- ERRORE di compilazione

📋 Convenzioni per le costanti

Per convenzione i nomi delle costanti si scrivono interamente in MAIUSCOLO con underscore come separatore, per distinguerle facilmente dalle variabili.

✓ MAX_VALORE — nome costante corretto
✓ VELOCITA_LUCE — leggibile, descrittivo
✗ maxValore — sembra una variabile normale

⚖️ Variabile vs Costante

VARIABILE
  • Valore modificabile
  • int eta = 20;
  • Può essere riassegnata
COSTANTE
  • Valore fisso
  • const int MAX = 30;
  • Errore se modificata
💡 Usare const ogni volta che un valore non deve cambiare: protegge da errori accidentali e rende il codice più leggibile, comunicando l'intenzione del programmatore.
