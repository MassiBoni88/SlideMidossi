Scope delle Variabili

01 / 04

📌 Cos'è lo scope?

Lo scope (o visibilità) di una variabile è la regione del programma in cui quella variabile esiste ed è accessibile.

🌍 Variabile Globale

  • Dichiarata fuori da qualsiasi funzione
  • Visibile in tutto il file
  • Esiste per tutta la durata del programma

📦 Variabile Locale

  • Dichiarata dentro una funzione o un blocco {}
  • Visibile solo in quel blocco
  • Distrutta quando il blocco termina
globale
int contatore = 0;   // visibile ovunque
locale → main()
int x = 5;   // solo in main
locale → blocco if
int y = 10;   // solo nell'if
locale → funzione foo()
int x = 99;   // diversa dalla x di main!
⚠️ Una variabile locale può avere lo stesso nome di una globale: la locale ha la precedenza nel suo scope.

Scope — Esempio Pratico

02 / 04 
// variabile GLOBALE
int x = 100;

void mostra() {
    // x globale: visibile qui
    cout << "mostra: x = " << x << endl;
}

int main() {
    int x = 5; // locale: nasconde la globale
    cout << "main: x = " << x << endl;

    {
        int x = 42; // locale al blocco
        cout << "blocco: x = " << x << endl;
    } // x=42 viene distrutta qui

    mostra(); // stampa la x globale
    return 0;
}

📤 Output

main:  x = 5
blocco: x = 42
mostra: x = 100
✅ Regola d'oro: C++ cerca la variabile prima nel blocco più interno, poi risale verso l'esterno (blocco → funzione → globale).
❌ Usare troppe variabili globali è cattiva pratica: rendono il codice difficile da leggere e da debuggare.

Scope — Parametri e Funzioni

03 / 04 
int somma(int a, int b) {
    // 'a' e 'b' sono LOCALI alla funzione
    int risultato = a + b;
    return risultato;
} // a, b, risultato vengono distrutti

int main() {
    int x = 3, y = 7;
    int tot = somma(x, y);

    // cout << risultato; // ERRORE!
    // risultato non esiste qui

    cout << tot << endl; // OK: 10
    return 0;
}

🧠 Da ricordare

  • I parametri di una funzione sono variabili locali a quella funzione
  • Le variabili locali nascono quando la funzione viene chiamata e muoiono quando ritorna
  • Non è possibile accedere a variabili locali di un'altra funzione
💡 Le funzioni comunicano tramite parametri e valori di ritorno, non tramite variabili condivise. Questo è il design corretto!

Scope — Variabili static

04 / 04 
void conta() {
    static int n = 0; // inizializzata UNA sola volta
    n++;
    cout << "Chiamata n. " << n << endl;
}

int main() {
    conta(); // Chiamata n. 1
    conta(); // Chiamata n. 2
    conta(); // Chiamata n. 3
    return 0;
}

⚡ static locale

  • Scope locale: visibile solo nella funzione
  • Lifetime globale: il valore viene conservato tra una chiamata e l'altra
  • Inizializzata una sola volta al primo accesso
⚠️ Una variabile static ha scope locale ma durata globale: una combinazione speciale, utile ma da usare con attenzione.

Overloading delle Funzioni

01 / 04

📌 Cos'è l'overloading?

In C++ è possibile definire più funzioni con lo stesso nome, purché abbiano parametri diversi (numero o tipo). Il compilatore sceglie quella giusta automaticamente.

✅ Cosa può differire

  • Il numero di parametri
  • Il tipo dei parametri
  • L'ordine dei tipi dei parametri

❌ Cosa NON basta

  • Cambiare solo il tipo di ritorno non è sufficiente
  • Cambiare solo il nome dei parametri non è sufficiente

📋 Esempio — funzione area()

FirmaRitornaUso
area(int l) int Quadrato di lato l
area(int b, int h) int Rettangolo
area(double r) double Cerchio (raggio r)
area(double b, double h) double Triangolo (b×h/2)
💡 L'overloading rende il codice più leggibile: non serve inventare nomi come area_quad, area_rett…

Overloading — Codice

02 / 04 
// versione 1: quadrato
int area(int lato) {
    return lato * lato;
}

// versione 2: rettangolo
int area(int base, int altezza) {
    return base * altezza;
}

// versione 3: cerchio
double area(double raggio) {
    return 3.14159 * raggio * raggio;
}

int main() {
    cout << area(5)       << endl; // 25
    cout << area(4, 6)    << endl; // 24
    cout << area(3.0)     << endl; // 28.27...
}

🔍 Come sceglie il compilatore?

Il compilatore analizza la firma della chiamata (numero e tipo degli argomenti) e sceglie la versione più adatta.

  • area(5) → un int → versione 1
  • area(4, 6) → due int → versione 2
  • area(3.0) → un double → versione 3
🧠 Questo meccanismo è detto risoluzione dell'overloading (overload resolution) ed avviene a compile time, non a runtime.

Parametri con Valore di Default

03 / 04 
// parametro 'sep' ha valore di default
void stampa(int a, int b,
           char sep = '+') {
    cout << a << sep << b << endl;
}

int main() {
    stampa(3, 5);        // 3+5  (usa default)
    stampa(3, 5, '-');   // 3-5  (sovrascrive)
    stampa(3, 5, '*');   // 3*5
    return 0;
}

// ATTENZIONE: i default vanno sempre
// dai parametri di destra verso sinistra!
// int f(int a=1, int b) → ERRORE!
// int f(int a, int b=2) → OK

📌 Regole dei parametri default

  • I valori di default si specificano nel prototipo (o nella definizione)
  • Devono essere gli ultimi nella lista dei parametri
  • Il chiamante può omettere quegli argomenti
⚠️ Attenzione a non creare ambiguità mescolando overloading e default: il compilatore potrebbe non sapere quale versione scegliere.

Overloading — Esercizio

04 / 04
✏️ Scrivi una funzione massimo() con overloading per i seguenti casi:

Versioni richieste

  • massimo(int a, int b) → maggiore tra due interi
  • massimo(double a, double b) → maggiore tra due reali
  • massimo(int a, int b, int c) → maggiore tra tre interi
// Esempio di chiamate attese:
int main() {
    cout << massimo(3, 7);
    // → 7

    cout << massimo(2.5, 1.8);
    // → 2.5

    cout << massimo(4, 9, 2);
    // → 9
}
💡 Bonus: la versione a tre parametri può richiamare internamente la versione a due parametri!

Riepilogo

📦 Scope

  • Locale: esiste solo nel blocco in cui è dichiarata
  • Globale: visibile in tutto il file
  • Locale nasconde globale con lo stesso nome
  • static: locale ma persiste tra le chiamate

🔁 Overloading

  • Stessa funzione, firme diverse
  • Si differenzia per numero e tipo dei parametri
  • Il compilatore sceglie a compile time
  • I valori default rendono parametri opzionali
