VLAN — Virtual LAN

L1 · 03/15

Cos'è una VLAN

Una VLAN è una rete locale virtuale che raggruppa logicamente porte di uno switch indipendentemente dalla posizione fisica degli host. Ogni VLAN costituisce un broadcast domain separato.

Vantaggi

  • Sicurezza: isolamento del traffico tra reparti
  • Performance: riduzione del dominio di broadcast
  • Flessibilità: riorganizzazione senza cablaggio fisico
  • Riduzione dei costi: un solo switch fisico per più reti
⚠️ Per comunicare tra VLAN diverse è necessario un router o uno switch L3 (Inter-VLAN routing). Due host su VLAN diverse non possono comunicare direttamente.

Porte Access e Trunk

  • Porta Access: appartiene a una sola VLAN; il frame viaggia senza tag
  • Porta Trunk: trasporta frame di più VLAN simultaneamente usando il tag 802.1Q
  • Le porte trunk collegano switch tra loro o switch a router

Tipi di VLAN

  • VLAN di gestione: accesso amministrativo agli switch (di default VLAN 1)
  • VLAN dati: traffico utente (voce, dati, video)
  • Native VLAN: su trunk 802.1Q, il traffico non taggato appartiene alla native VLAN

Inter-VLAN Routing — Router-on-a-Stick

Un'unica interfaccia fisica del router viene divisa in sottointerfacce (subinterface), una per ogni VLAN. Ogni subinterface è configurata con l'incapsulamento 802.1Q e un indirizzo IP nella subnet della VLAN corrispondente.

Switch Layer 3 (SVI)

Uno switch L3 crea una Switched Virtual Interface (SVI) per ogni VLAN con un indirizzo IP. Il routing avviene all'interno dello switch stesso, senza bisogno di un router esterno, con prestazioni superiori.

💡 Il VID 0 è riservato (priorità solo), il VID 1 è la VLAN nativa/default, il VID 4095 è riservato. In pratica si usano VID da 2 a 1001 (range normale) e da 1006 a 4094 (range esteso su switch Cisco).

Spanning Tree Protocol (STP)

L2 · 05/15

Il problema dei loop

In una rete con switch ridondanti si possono formare loop di livello 2. Un frame broadcast entra in un loop infinito: broadcast storm, che satura la banda e blocca la rete.

STP — IEEE 802.1D

STP elimina i loop bloccando logicamente le porte ridondanti, mantenendo un unico albero spanning attivo. Le porte bloccate rimangono in standby e si attivano automaticamente in caso di guasto.

  • Elezione del Root Bridge (Bridge ID più basso)
  • Calcolo del percorso di costo minimo verso il Root Bridge
  • Blocco delle porte ridondanti (Blocking state)

RSTP — Rapid Spanning Tree

L2 · 06/15

IEEE 802.1w — RSTP

RSTP riduce il tempo di convergenza da 30–50 s a meno di 1 secondo grazie a un meccanismo di negoziazione rapida tra switch adiacenti (proposta/accordo).

Ruoli delle porte RSTP

  • Root Port: porta con percorso di costo minimo verso il Root Bridge
  • Designated Port: porta "eletta" per ogni segmento di rete
  • Alternate Port: backup per la Root Port (sostituisce Blocking in STP)
  • Backup Port: backup per la Designated Port sullo stesso segmento
  • Edge Port: collegata a un host terminale (equivalente a PortFast)

MSTP — IEEE 802.1s

Multiple STP mappa più VLAN su istanze STP distinte. Consente di bilanciare il carico su link ridondanti: le VLAN 10–20 usano il path A come root, le VLAN 30–40 usano il path B. Ideale per reti con molte VLAN.

PortFast e BPDU Guard

  • PortFast: salta gli stati Listening/Learning su porte edge, l'host è operativo subito
  • BPDU Guard: se una porta PortFast riceve un BPDU (da un switch), la disabilita automaticamente → protezione da loop accidentali
✅ RSTP è retrocompatibile con STP 802.1D. Su reti Cisco si configura con spanning-tree mode rapid-pvst (Rapid Per-VLAN STP).

Firewall — Tipologie e Architetture

L3 · 07/15

1 — Packet Filter (Stateless)

Analizza ogni pacchetto indipendentemente basandosi su: IP sorgente/destinazione, porta, protocollo. Veloce ma non conosce lo stato della connessione. Opera a livello 3–4 del modello OSI.

2 — Stateful Inspection

Mantiene una tabella di stato delle connessioni TCP/UDP attive. Controlla che i pacchetti in ingresso appartengano a una sessione legittima avviata dall'interno. Standard nei firewall moderni.

3 — Application Gateway (Proxy FW)

Opera a livello 7 (applicazione). Analizza il contenuto dei protocolli (HTTP, FTP, DNS). Il client non comunica mai direttamente con il server esterno: il firewall fa da intermediario.

4 — NGFW (Next Generation)

Integra packet filter, stateful, IPS, deep packet inspection (DPI), controllo applicativo (riconosce Facebook, Skype), filtraggio URL e antivirus. Soluzione unificata UTM.

Architettura DMZ

La DMZ (DeMilitarized Zone) è una rete intermedia tra Internet e la rete interna. Ospita i server pubblici (web, mail, DNS). Due firewall: uno tra Internet e DMZ, uno tra DMZ e rete interna.

🛡️ Regola di default: i firewall applicano una politica "deny all" implicita — tutto il traffico non esplicitamente permesso viene bloccato.

IDS e IPS

L3 · 08/15

IDS — Intrusion Detection System

Monitora il traffico di rete e segnala attività sospette (alert, log). Non blocca il traffico — è un sensore passivo. Posizionato in modalità promiscua (tap/SPAN port): vede il traffico ma non è nel percorso dei dati.

  • Falsi positivi: traffico legittimo segnalato come attacco
  • Falsi negativi: attacco non rilevato

IPS — Intrusion Prevention System

Monitora il traffico inline (è nel percorso dei dati) e può bloccare attivamente il traffico malevolo in tempo reale: drop del pacchetto, reset della connessione, blocco dell'IP sorgente.

Metodi di rilevamento

  • Signature-based: confronta il traffico con firme di attacchi noti (aggiornamento periodico del database)
  • Anomaly-based: stabilisce una baseline del traffico normale; segnala le deviazioni statistiche
  • Policy-based: controlla il rispetto di politiche di sicurezza predefinite

HIDS vs NIDS

  • HIDS (Host-based): installato sul singolo host; monitora log, processi, file system
  • NIDS (Network-based): monitora il traffico su un segmento di rete; unico sensore per molti host
💡 Nei NGFW, IDS e IPS sono integrati nel sistema. Un IPS in linea introduce una latenza minima ma garantisce protezione attiva; un IDS fuori linea non ha impatto sulle prestazioni.

ACL — Access Control List

L4 · 09/15

Cos'è una ACL

Una ACL è un elenco ordinato di istruzioni (ACE — Access Control Entry) che un router Cisco usa per filtrare il traffico IP su un'interfaccia. Ogni regola può permit (lasciar passare) o deny (bloccare) i pacchetti in base ai criteri specificati.

Regole fondamentali

  • Le regole sono valutate in ordine, dall'alto verso il basso
  • Al primo match la regola viene applicata e si smette di leggere la lista
  • Alla fine di ogni ACL c'è un implicit deny any (tutto ciò che non è esplicitamente permesso è negato)
  • Una ACL vuota nega tutto (solo implicit deny)
📍 Posizionamento: ACL Standard → il più vicino possibile alla destinazione; ACL Estesa → il più vicino possibile alla sorgente.

Wildcard Mask

La wildcard mask è il complemento della subnet mask. Un bit a 0 = controlla quel bit; bit a 1 = ignora quel bit.

  • Subnet /24 → mask 255.255.255.0 → wildcard 0.0.0.255
  • Subnet /16 → mask 255.255.0.0 → wildcard 0.0.255.255
  • host 10.1.1.1 = 10.1.1.1 0.0.0.0 (match esatto)
  • any = 0.0.0.0 255.255.255.255 (qualsiasi IP)

Tipi di ACL Cisco

  • Standard (1–99, 1300–1999): filtra solo per IP sorgente
  • Estesa (100–199, 2000–2699): filtra per IP src/dst, protocollo, porta
  • Named: ACL standard o estesa identificata da un nome invece che da un numero

ACL Standard — Sintassi ed Esempi

L4 · 10/15

Sintassi (numerata)

! Creazione regola
Router(config)# access-list <1-99> {permit|deny} <source> [wildcard]
Router(config)# access-list <1-99> {permit|deny} host <ip>
Router(config)# access-list <1-99> {permit|deny} any

! Applicazione sull'interfaccia
Router(config-if)# ip access-group <acl-number> {in|out}

! Verifica
Router# show access-lists
Router# show ip interface GigabitEthernet0/0

Esempio 1 — Blocca un host

! Nega solo 192.168.1.10, permette tutto il resto
Router(config)# access-list 10 deny   host 192.168.1.10
Router(config)# access-list 10 permit any

Router(config-if)# ip access-group 10 in

Esempio 2 — Permetti solo una subnet

! Permette solo 172.16.0.0/16, blocca tutto il resto
Router(config)# access-list 20 permit 172.16.0.0 0.0.255.255
Router(config)# access-list 20 deny   any
! (l'ultima riga è ridondante: c'è già l'implicit deny)

Router(config-if)# ip access-group 20 out
📍 Le ACL Standard filtrano solo in base all'IP sorgente: posizionale vicino alla destinazione per evitare di bloccare il traffico verso reti che non si vuole filtrare.

ACL Estese — Sintassi ed Esempi

L4 · 11/15

Sintassi completa (numerata)

Router(config)# access-list <100-199> {permit|deny} <proto>
                  <src> <src-wc> [<dst> <dst-wc>]
                  [eq|lt|gt|neq|range <port>]

  proto: ip | tcp | udp | icmp | ospf | eigrp | …
  eq 80 = porta 80   |   range 20 21 = porte 20 e 21
  established = solo TCP con flag ACK o RST (connessioni già avviate)

Esempio 1 — Blocca Telnet verso una rete

! Blocca Telnet (tcp/23) verso 192.168.1.0/24
! da qualsiasi sorgente, permette tutto il resto
Router(config)# access-list 101 deny   tcp any
                  192.168.1.0 0.0.0.255 eq 23
Router(config)# access-list 101 permit ip  any any

Esempio 2 — Permetti HTTP/HTTPS a un server

! Permette web (80,443) verso server 10.0.0.100
Router(config)# access-list 102 permit tcp any
                  host 10.0.0.100 eq 80
Router(config)# access-list 102 permit tcp any
                  host 10.0.0.100 eq 443
Router(config)# access-list 102 deny   ip  any any

Named ACL — Sintassi ed Esempio

! Named ACL estesa: più leggibile, permette cancellazione singole righe
Router(config)# ip access-list extended BLOCCA_FTP
Router(config-ext-nacl)#  deny   tcp 192.168.0.0 0.0.0.255 any eq 21
Router(config-ext-nacl)#  deny   tcp 192.168.0.0 0.0.0.255 any eq 20
Router(config-ext-nacl)#  permit ip  any any

Router(config-if)# ip access-group BLOCCA_FTP in

NAT e PAT

L5 · 12/15

NAT — Network Address Translation

Il NAT traduce gli indirizzi IP privati (RFC 1918) in indirizzi IP pubblici durante l'inoltro dei pacchetti. Obiettivi: conservare lo spazio di indirizzamento IPv4 e nascondere la topologia interna.

Tipi di NAT

  • Static NAT: mapping 1:1 fisso tra IP privato e IP pubblico (es. server DMZ)
  • Dynamic NAT: pool di IP pubblici assegnati dinamicamente agli host interni
  • PAT / NAT Overload: molti IP privati → 1 IP pubblico, distinti dalla porta sorgente (TCP/UDP)
🌐 Il PAT (Port Address Translation) è il NAT più usato nelle reti SOHO: tutti gli host condividono un unico IP pubblico, differenziati dalla porta di origine.

Limiti del NAT

  • Rompe il modello end-to-end di IP
  • Problemi con protocolli che incorporano IP nel payload (FTP, SIP)
  • Aumenta la latenza (lookup della tabella per ogni pacchetto)
  • Con IPv6 il NAT non è necessario (spazio di indirizzamento enorme)

Proxy Server

L5 · 13/15

Forward Proxy

Posizionato tra i client della rete interna e Internet. Il client invia le richieste al proxy, che le inoltra verso il server esterno. Il server vede solo l'IP del proxy, non quello del client.

  • Cache: le risorse richieste più volte vengono salvate localmente → riduzione del traffico e latenza
  • Filtraggio: blocco di URL e categorie di siti (parental control, policy aziendale)
  • Autenticazione: solo utenti autenticati possono navigare
  • Log: registrazione di tutta l'attività web degli utenti
🔍 Il proxy più diffuso in ambienti Linux è Squid. Opera sulla porta 3128 (HTTP) e può fare HTTPS inspection con certificate spoofing.

Reverse Proxy

Posizionato davanti ai server web interni. I client esterni credono di parlare direttamente con il server, ma in realtà comunicano con il reverse proxy che inoltra le richieste ai server backend.

  • Load balancing: distribuisce le richieste su più server
  • SSL termination: gestisce TLS/HTTPS, i server backend lavorano in HTTP
  • WAF (Web Application Firewall): filtra richieste malevole (SQL injection, XSS)
  • Cache: riduce il carico sui server backend

Proxy vs NAT — Differenze

  • NAT: opera a livello 3/4; trasparente alle applicazioni; non legge il contenuto
  • Proxy: opera a livello 7; conosce il protocollo applicativo; può ispezionare/modificare il contenuto
  • Il proxy richiede configurazione esplicita nel browser o un proxy trasparente

Riepilogo — Unità 2

🟢 L1 — VLAN

  • Segmentazione logica indipendente dalla topologia fisica
  • IEEE 802.1Q: tag 4 byte con VID (12 bit, 0–4095) su porte trunk
  • Porta Access (1 VLAN, no tag) vs porta Trunk (più VLAN, tagged)
  • Inter-VLAN: Router-on-a-stick (subinterface) o Switch L3 (SVI)

🔵 L2 — STP / RSTP

  • STP (802.1D): elimina loop bloccando porte ridondanti, convergenza ~30–50 s
  • Root Bridge: switch con BID più basso (priorità + MAC)
  • RSTP (802.1w): convergenza <1 s, ruoli Alternate/Backup/Edge
  • PortFast + BPDU Guard: porte edge veloci e protette

🔴 L3 — Firewall e IDS/IPS

  • Tipologie: packet filter → stateful → application proxy → NGFW
  • DMZ: zona intermedia per server pubblici
  • IDS: passivo (fuori linea), IPS: attivo (in linea)
  • Rilevamento: signature-based, anomaly-based, policy-based

🟡 L4 — ACL Cisco

  • Valutazione top-down, primo match → applica regola, implicit deny finale
  • Standard (1-99): solo IP sorgente → posiziona vicino alla destinazione
  • Estesa (100-199): src, dst, protocollo, porta → posiziona vicino alla sorgente
  • Named ACL: nome invece di numero, possibile cancellare singole righe
  • Wildcard: 0=controlla, 1=ignora. host X = X 0.0.0.0, any = 0.0.0.0 255.255.255.255

🟢 L5 — NAT, PAT e Proxy

  • Static NAT: 1:1 fisso; Dynamic NAT: pool; PAT: molti:1 con porte diverse
  • Tabella NAT: IP_priv:porta_src ↔ IP_pub:porta_NAT ↔ destinazione
  • Forward Proxy: client→proxy→Internet (cache, filtri, log)
  • Reverse Proxy: Internet→proxy→server (load balancing, SSL, WAF)
