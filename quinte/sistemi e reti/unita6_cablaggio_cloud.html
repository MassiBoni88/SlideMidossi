Progettare la Struttura Fisica della Rete

Lezione 1 · 01/02

🏢 3 Macrofattori della rete aziendale

  • Struttura fisica della LAN: topologia, mezzi trasmissivi, apparati di rete
  • Virtualizzazione delle risorse hardware e software
  • Cloud computing: utilizzo di risorse e servizi IT in remoto

⭐ Topologia a stella estesa

La struttura più usata nelle reti LAN aziendali — detta anche stella gerarchica. Collega tra loro più topologie a stella.

✅ Vantaggi della topologia a stella estesa

  • Fault-tolerance: il guasto di un canale o di un'interfaccia non compromette il funzionamento generale della rete
  • Flessibilità ed espandibilità: lo spostamento di un host da un punto a un altro della rete o l'inserimento di uno nuovo non richiedono il fermo della rete
⚠️ Punto critico: la topologia è vulnerabile al livello degli switch (centri stella). Se lo switch che svolge questo ruolo si guasta, la rete, o parte di essa, smette di funzionare.
💡 Rispetto alla topologia a bus o ad anello, la stella estesa offre notevoli vantaggi in termini di fault-tolerance e flessibilità, a fronte di un maggior numero di cavi.

Il Cablaggio Strutturato — Standard ISO/IEC 11801

Lezione 1 · 02/02
📌 Il cablaggio strutturato è l'insieme delle regole per collocare i nodi e collegarli, indicando dove tirare i cavi o inserire i punti di accesso wireless. Lo standard di riferimento è ISO/IEC 11801.
GERARCHIA ISO/IEC 11801 — DISTANZE MASSIME
CD — Campus Distributor  (centro stella di comprensorio)
max 1500 m
BD — Building Distributor  (centro stella di edificio)
max 500 m
FD — Floor Distributor      (centro stella di piano)
max 90 m
TO — Telecommunication Outlet  (presa utente)
max 5 m

🔌 Tipi di cablaggio

  • Verticale (VCC) — Vertical Cross-Connect: collega CD → BD → FD.
    Usa fibra ottica nelle dorsali (prestazioni elevate, anche per distanze brevi)
  • Orizzontale (HCC) — Horizontal Cross-Connect: collega FD → TO.
    Usa cavi UTP o STP (raramente fibra, costi maggiori)

🏗️ Schema tipico su 2 edifici

Edificio 1: 1 piano — un FD che funge anche da BD, al centro c'è il CD (comprensorio).
Edificio 2: 2 piani — due FD collegati al BD di edificio.

Le prese utente (TO) sono nei punti del piano dove si trovano postazioni utente o stampanti di rete.

💡 Il cablaggio orizzontale usa UTP/STP. Nelle dorsali si preferisce la fibra ottica, anche per distanze ridotte, per le elevate prestazioni.

Progettare la Collocazione dei Server

Lezione 2 · 01/03
🖥️ I server vengono identificati in base al servizio che erogano. Le piccole aziende usano macchine standalone; le medie/grandi allestiscono data center interni o si appoggiano a server farm gestite da data center esterni.

🗂️ Storage & Accesso dati

  • File Server: condivisione file in rete
  • Database Server: gestione banche dati
  • FTP Server: cartelle pubbliche o autenticate
  • Print Server: stampanti condivise in rete
  • Grid computing: elaborazione distribuita di grandi quantità di dati

🌐 Web, App & Comunicazione

  • Web Server: ospita siti (HTTP)
  • Application Server: app web condivise
  • Mail Server: posta elettronica
  • Proxy Server: cache web + ACL
  • Game Server: giochi multiplayer online

🔐 Rete, Sicurezza & Virtualizzazione

  • DHCP Server: assegnazione automatica IP
  • DNS Server: risoluzione nomi di dominio
  • AD Server: Active Directory (Domain Controller)
  • VPN Server: accesso LAN da remoto
  • VNC Server: desktop remoto
  • Server di autenticazione: accesso ad altre risorse

Data Center e Server Farm

Lezione 2 · 02/03

🖥️ Forme fisiche dei server

TOWER Case verticale simile al PC. Autonomo (standalone), capacità storage RAID. Voluminoso e rumoroso. Usato per filiali di piccole aziende
RACK Montato su armadi rack. Compatto, semplifica i cablaggi, facile scalabilità. Ideale nei data center aziendali
BLADE Lame intercambiabili in uno chassis condiviso. Ottimizza spazio, una singola lama può simulare N macchine virtuali. Soluzione tipica con virtualizzazione
💡 La virtualizzazione rappresenta una soluzione tipicamente associata ai server blade, unita al consolidamento di diversi sistemi di storage.

🏢 Il Data Center — CED

Centro Elaborazione Dati: area attrezzata per il trattamento e l'archiviazione dati, collocato nello stesso locale tecnico del centro stella di comprensorio o di edificio.

  • Interno: controllo diretto, intervento rapido, costi fissi elevati (affitto, hardware, aggiornamenti, personale)
  • Esterno: delegato a ISP specializzati. Sicurezza garantita, riduce costi di manutenzione. Ideale per applicazioni web

🏭 Le Server Farm

Aree fisiche, normalmente nel sottosuolo, con decine/centinaia/migliaia di server. Caratteristiche obbligatorie:

  • Sicurezza fisica + sistemi anti-intrusione
  • Alimentazione ridondante (UPS, gruppi di continuità)
  • Impianto di condizionamento dedicato (bassa temperatura)
  • Connettività Internet stabile e garantita
  • Firewall e protezione logica delle macchine

I Servizi offerti dalle Server Farm

Lezione 2 · 03/03

🌐 Hosting

Il server è di proprietà del provider. Si ospita il sito/app web su server del fornitore.

  • Gratuito: dominio condiviso con l'ISP, solo HTML statico, banda condivisa tra molti utenti, banner pubblicitari obbligatori
  • A pagamento: dominio personalizzato, email illimitate, database, scripting, banda garantita, statistiche traffico

📦 Colocation in Housing

Il server è di proprietà dell'azienda, localizzato fisicamente nella server farm del data center. L'azienda acquista e gestisce il server; paga al provider solo l'affitto dello spazio, alimentazione e rete.

PRO: proprietà hardware, controllo totale SW
CONTRO: costo startup, interventi da concordare

🔑 Server Dedicati

Il data center mette a disposizione server dedicati ad uso esclusivo del cliente. Il cliente noleggia il server e lo amministra da remoto senza preoccuparsi di upgrade hardware o riparazioni (a carico del provider).

PRO: aggiornamenti sicurezza delegati, nessun costo startup
CONTRO: canone mensile elevato per app critiche

☁️ Server Virtuali — VPS

Virtual Private Server: il provider offre una macchina virtuale su un server fisico condiviso. Appare come una macchina dedicata, ma le spese hardware sono divise tra più clienti.

  • Costi bassi — ideale per PMI e startup
  • Risorse CPU/RAM garantite (buffer garantito)
  • Separazione netta tra ambienti di aziende diverse

La Virtualizzazione dei Server — Sezione 3.1

Lezione 3 · 01/01
📌 La Server Virtualization permette di creare su un singolo server più virtual machine in contemporanea, che condividono le risorse hardware e su cui girano diversi sistemi operativi e applicativi client.

🏗️ Architettura Host / Guest

App     App     App
OS guest 1  |  OS guest 2  |  OS guest N
Layer di virtualizzazione — Hypervisor
OS host (layer inferiore — sistema operativo ordinario)
Hardware fisico (server)
  • Host: il SO ospitante, crea partizioni per i guest; fornisce caratteristiche fino al BIOS
  • Guest: i SO ospiti che pensano di avere tutta la macchina per sé

✅ Vantaggi della Virtualizzazione

  • Riduzione dei costi di implementazione e gestione (consolidamento hardware)
  • Riduzione del consumo energetico dell'intero data center
  • Allocazione dinamica delle risorse
  • Riduzione drastica dei tempi di messa in opera di nuovi sistemi
  • Isolamento da problemi di sistema operativo o applicativo
  • Semplificazione della gestione risorse eterogenee
  • Facilitazione di testing e debugging in ambienti controllati
  • Gestione dell'evoluzione tecnologica: migrazione verso nuovo hardware senza reinstallare tutto
🔄 Se l'hardware diventa obsoleto, basta reinstallare il layer emulato e ripristinare i file delle macchine virtuali — senza reinstallare tutto il sistema.

La Virtualizzazione dei Software

Lezione 4 · 01/01

🖥️ Virtualizzazione del Sistema Operativo

Prima di virtualizzare occorre procurarsi un'immagine ISO del SO da installare (copia esatta del CD/DVD di installazione).

VirtualBox Oracle — gratuito, open source, multipiattaforma (Win/Mac/Linux). Supporta iSCSI come dischi virtuali
VMware Workstation (Win/Linux), Fusion (Mac). Versione Player gratuita, versione Pro avanzata
Parallels Solo Mac — eccellente integrazione macOS, permette di eseguire Windows e Linux su Mac. A pagamento (versione prova gratuita)
📌 Un file ISO è genericamente detto immagine — è un unico file che contiene esattamente gli stessi file e la struttura di un CD/DVD.

📦 Virtualizzazione delle Applicazioni

Invece di creare una VM completa, si virtualizzano solo le applicazioni creando una "bolla" isolata che non modifica il sistema operativo.

SVS (Altiris/Symantec) — definisce layer per ogni app, attivabili/disattivabili in qualsiasi momento
VMware / Citrix Virtual Apps — distribuisce app come stream di bit on demand
Microsoft App-V — per Windows 10, incapsula app in file eseguibili (bolla)

✅ Benefici principali

  • Gestione centralizzata delle applicazioni
  • Protezione app e dati nel data center
  • Riduzione del TCO (Total Cost of Ownership)
  • Semplicità di aggiornamento e manutenzione
  • Isolamento di applicazioni conflittuali o pericolose
  • Controllo completo accesso e attività dell'utente

Le Soluzioni Cloud

Lezione 6 · 01/03
☁️ Il cloud computing è un insieme di tecnologie informatiche che permettono l'utilizzo di risorse hardware e software, virtualizzate e distribuite in remoto, in una tipica architettura Client/Server. Ma soprattutto è la possibilità di distribuire e utilizzare servizi IT attraverso il web.

🎯 Perché il cloud?

  • Abbatte i costi degli investimenti nei CED aziendali e pubblici
  • Accesso rapido a enormi risorse (calcolo, memorizzazione) in modo virtuale
  • Pagamento in base all'effettivo utilizzo (no costi fissi infrastruttura)
  • Supera le barriere di costo e complessità dell'IT tradizionale
  • Accessibile alle PMI e startup con investimenti limitati

🔷 I 4 tipi di Cloud

PRIVATE Ambiente informatico interno all'azienda; virtualizza risorse e standardizza la gestione. Controllo diretto su sicurezza e infrastruttura
PUBLIC Fornito da provider esterno (AWS, Azure, Google Cloud) che ne garantisce efficienza e sicurezza tramite standard
COMMUNITY Condiviso tra organizzazioni con interesse comune (es. database medico condiviso tra tutti gli ospedali di una nazione)
HYBRID Unisce private + public: sfrutta risorse interne e le estende al cloud pubblico on demand. Trattato nella Lezione 7
🔗 È possibile anche un Multi-cloud computing: soluzione composta da due o più cloud distinti (pubblici o privati o entrambi) per massimizzare flessibilità e ridondanza.

Architettura Cloud e Modelli di Servizio (XaaS)

Lezione 6 · 02/03

🏗️ Componenti di una rete cloud

  • Cloud client: hardware (qualsiasi dispositivo connesso a Internet) + software (browser) per fruire del cloud
  • Cloud application: architetture software che eliminano l'installazione locale (es. Gmail, Office 365 — SaaS)
  • Cloud platform: stack di elaborazione e memorizzazione approntate sulle esigenze dell'utente (es. Jelastic di Aruba — PaaS)
  • Cloud storage: memorizzazione dati come servizio calcolato per GB/mese
  • Cloud infrastructure: reti di computer virtuali messe a disposizione (IaaS)
👥 I 3 ruoli fondamentali: Cloud provider (possiede le risorse), Cloud user (fruitore finale — rischio privacy), Cloud vendor (cliente amministratore che rivende servizi cloud).

📦 I Modelli di Servizio — XaaS

SaaS

Software via browser senza installazione. Es. Gmail, Office 365

DaaS

Solo i dati via web, accessibili da qualsiasi app come se fossero in locale

HaaS

Dati inviati in remoto, elaborati da hardware cloud, risultati restituiti

PaaS

Piattaforma software completa (librerie, runtime, DB) eseguita in remoto

IaaS — Infrastructure as a Service

Utilizzo di risorse hardware in remoto. Simile al grid computing, ma le risorse sono assegnate on demand.

Leader: Amazon Web Services · Windows Azure · Google Cloud

Il Cloud nella Pubblica Amministrazione

Lezione 6 · 03/03

🏛️ Perché il cloud per la PA?

Il modello a consumo o ad abbonamento (modello a servizio) libera la PA dalle attività di manutenzione, aggiornamento e distribuzione, rapportando il pagamento all'erogazione di un servizio.

  • Alcune PA realizzano private cloud interni con data center propri
  • Altre affidano tutto a cloud provider esterni con propri data center
  • Tramite il cloud i governi offrono servizi pubblici più efficienti: eHealth, eGovernment, eJustice

Le Soluzioni Ibride: Hybrid Cloud

Lezione 7 · 01/02
🔗 Una soluzione Hybrid cloud unisce private cloud e public cloud. Consente di sfruttare le risorse aziendali interne (On-Premise) e di estendere il proprio data center fino al cloud pubblico (Off-Premise), collegati tramite VPN IPsec.

On-Premise
(infrastruttura aziendale)
IPsec VPN
⇄
PUBLIC CLOUD
Off-Premise
(cloud provider esterno)

L'architettura comprende sia la struttura di proprietà dell'azienda (On-Premise) sia quella di proprietà del provider esterno (Off-Premise).

✅ Possibilità dell'Hybrid Cloud

  • Spostare carichi di lavoro dal data center aziendale al provider di hosting
  • Creare applicazioni ibride su App Server che sfruttano sia risorse locali sia cloud
  • Utilizzare opzioni di Warm Disaster Recovery più efficienti e convenienti

⚖️ Load Balancer — Soluzione bilanciata

I software di gestione consentono di bilanciare le richieste tra le due nuvole, spostando le risorse dove sono più utili.

Private cloud — Load balancer · App servers · Master DB · Slave DB · Object storage
Public cloud  — Load balancer · App services · Slave DB · Object storage

Questa è la soluzione Warm Disaster Recovery: il public cloud funge da backup attivo.

🔄 Il load balancer applica scalabilità in base alle esigenze specifiche dell'azienda, in modo completamente trasparente per l'utente finale.
🔧 I software per la gestione ibrida trasformano le varie piattaforme in un'unica entità che può essere gestita come un unico sistema.

L'Adozione del Hybrid Cloud

Lezione 7 · 02/02

📈 Perché scegliere l'Hybrid Cloud

  • I piani di adozione dell'Hybrid cloud sono in costante crescita
  • Le aziende affrontano l'IT in termini di pool di risorse fruibili attraverso infrastrutture pubbliche, private, dedicate e virtuali
  • Possibilità di spostare rapidamente il carico di lavoro dal private cloud al public cloud
  • Utilizzo del cloud come luogo di backup per il disaster recovery
  • Ottenere il giusto mix di prestazioni per il proprio business
📊 Secondo studi di Gartner Group, entro il 2021 il 90% delle aziende avrebbe implementato un modello di cloud ibrido.

🔐 Sfide di sicurezza

  • Sul public cloud i dati di più aziende sono condivisi: la sicurezza è una priorità assoluta
  • Non tutte le piattaforme di crittografia supportano sia public cloud che private cloud
  • Le aziende devono pianificare attentamente come il cloud scalerà in caso di aumento o diminuzione della richiesta di risorse

🔧 Complessità e pianificazione

L'Hybrid cloud è più complesso rispetto al solo public o private cloud. Richiede:

  • Pianificazione attenta delle strategie di scalabilità
  • Software di gestione ibrida (integrazione delle piattaforme)
  • Politiche di sicurezza unificate tra On-Premise e Off-Premise
  • Investimento in competenze specifiche del personale IT
